Newsletter Signup Page

A signup page focusing on forms and inputs.

Newsletter Signup Page

Technologies Used:

Parameters:

Create a form page from a client mockup using HTML and CSS.

Challenges:

HTML forms are quite robust, requiring many attributes and labels. It was important to make sure that everything was connected correctly. Additionally, inputs are not as easy to style as other elements. It took a lot of research to find out what parts of inputs can be styled. I attempted to use a neumorphism approach to the styling. It was both challenging and rewarding, but I'm sure that I can refactor the design to be even more dynamic. I plan to add validation to this form using JavaScript.

Takeaways:

I am still actively studying form elements and styling. I want to make sure forms I create in the future are accessible and use proper markup. In addition, while styling this form, I realized that user experience and accessibility must be prioritized. It was a great learning experience and a fun project.

Employee Directory

A directory using an API to generate random employee information.

Employee Directory

Technologies Used:

Parameters:

Create an interactive directory based on client mockup that displays information cards and detailed information modals.

Challenges:

This was my first attempt at using an API, using promises made it a lot easier. The biggest speedbump came when trying to wrap my head around pulling information from the API. Once I started thinking in terms of objects, it made more sense. I also ran into some trouble when creating the search function because I didn't fully understand the difference between static and live html lists. Now that I have a better understanding of these lists, I'll run into fewer issues in the future.

Takeaways:

Almost immediately after working on this project, I started to think of other tasks that I could use APIs to complete. There is something appealing to me about pulling data out of collections of objects. I am particularly interested in the practices of consolidation, efficiency, and using the right tool for the job. I think APIs offer a great way to hit those key areas.

Photo Gallery

A user-friendly, interactive photo gallery with lightbox and search functions.

Photo Gallery

Technologies Used:

Parameters:

Create an interactive photo gallery that allows search and uses lightbox to interact with photos.

Challenges:

There were two main challenges with this project. The first was understanding how to properly use libraries, plugins, and modules. I had some initial trouble getting the baguettebox plugin working becuase I wasn't sure how the plugin interacted with my own code. The project file structure needed some adjusting on my end. After learning that a plugin really is just that, it all made sense. The second challenge was creating the search function from scratch. It took some research, but I eventually found a tutorial that made sense. This was one of the first in-depth JavaScript projects I attemtped. Having learned more about DOM traversal, search functions are now easier to understand and implement.

Takeaways:

This project showed me that plugins and libraries are powerful, useful tools. Since completing this project, I have implemented several into other projects. However, while still learning, I'd like to use vanilla JS as much as possible.

React Scoreboard

A scoreboard with multiple functioning parts including a stopwatch and leader marker.

React Scoreboard

Technologies Used:

Parameters:

Create a scoreboard using React focusing on components and state.

Challenges:

React is an incredible framework. As a new developer, the myriad challenges posed by React were a bit overwhelming. I've gone through this tutorial several times to reinforce the concepts. The main challenge for me was understanding how to properly structure a React application. I know this will improve with more practice, as of right now, it’s still challenging. The syntax and component interaction were confusing for me, but are becoming clearer each time I build with React. Another minor issue was making sure my file structure was correct, especially since I wanted to deploy through Github pages. Fortunately, plenty of documentation exists about how to deploy on Github pages.

Takeaways:

React will be my go-to frontend development tool. Although it’s a tad to understand, the theory behind it makes sense to me. I like that React is built around breaking things down to their base components. Modular components, are easier to manipulate and improve over time. I find React really enjoyabe to work with for these reasons.

Journey

Yesterday

My coding journey began with a few online courses introducing HTML and CSS. I was blown away by what I could do, or more accurately, what I hoped I could do someday, with a few lines of well-written CSS. After whetting my appetite, I quickly moved on to JavaScript where my mind was re-blown. I've spent the bulk of my time studying these 3 langauges. I'm comfortable with and fond of all three.

Today

While I am still continuing to learn in the vanilla JS and CSS realm, I couldn't help but start to look for what was next. After doing some research, I decided that my next goal would be to take on the most popular JS framework, React. I had no idea what I was in for. There is so much to learn! In addition to React, I've set my sights on becoming more comfortable with Bootstrap, Tailwind, and TypeScript. These are my current, admittedly lofty, goals.

Tomorrow

The most exciting thing about the world of coding is the fact that there's always something new on the horizon. I'm currently looking forward to getting a handle on state management using Redux. I also can't wait to expand my abilities to the back end through node.js and start learning more about databases with mongoDB and data analysis with python. There is so much to learn and a limitless number of paths to follow.
